# Validation Policy and Current Evidence

The repository does not treat a manually copied descriptor checksum as permanent evidence. Protobuf descriptors, normalized module IR, checksums and diagnostic reports are reproducible build outputs generated by CI from the committed source.

## Required checks

- **Contract CI** — Buf build, canonical formatting, STANDARD lint and breaking compatibility against `main`.
- **Governance CI** — strict YAML parsing, JSON Schema validation, semantic ownership/dependency validation, normalized IR compilation, Python/Rust digest parity and architecture boundaries.
- **Rust CI** — workspace formatting, Clippy with warnings denied and all workspace tests.

## Contract validation

- Compiler paths: Buf and `grpc_tools.protoc`.
- Proto source files: 28 at the time this policy was updated.
- Descriptor generation: `python scripts/validate_contracts.py` or Contract CI.
- Descriptor location: temporary build/workflow artifact; not an authoritative committed file.
- Descriptor SHA-256: calculated on every run and recorded in the corresponding job output/artifact.
- Known protobuf warnings: none accepted. Warnings and lint exceptions must be explicit and narrowly scoped in `buf.yaml`.

## Module manifest validation

```text
module.yaml
→ strict authoring validation
→ normalized JSON IR
→ crm.cjson/v1 canonical bytes
→ SHA-256 digest
→ typed Rust ModuleManifest verification
```

Governance CI fails when the Python authoring compiler and Rust runtime representation disagree on accepted semantics or digest.

## Evidence rule

A statement that a check passes is valid only for the exact commit whose GitHub Actions run completed successfully. Production-readiness evidence additionally requires security, load, restore and disaster-recovery gates defined in `docs/IMPLEMENTATION_ROADMAP.md`.
